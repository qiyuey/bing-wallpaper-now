# Release Checklist

发布流程的补充规则。主流程见 `SKILL.md`，此文件仅在需要时读取。

## 红线规则

- 禁止通过抑制告警掩盖问题（如随意添加 `#[allow(...)]`）
- 禁止将无关改动、临时调试代码带入发布
- 不做破坏性 Git 操作（`reset --hard`、强推主分支）
- `make release` 失败时立即停止，不要手动补救

## CHANGELOG 写作规范

- 描述面向用户，不面向开发者
- 使用"新增 / 修复 / 改进"等用户可理解的语言
- 避免空泛描述（如"优化代码结构"不带具体改动说明）
- 避免不可验证的结论（如"大幅提升性能"但无数据）

## 发布后验证

- 确认 tag 指向正确的 commit（`git show <tag>`）
- 确认 CI/CD 任务已触发（GitHub Actions）
- 确认各平台构建产物正常生成
